## Требования

**ПО**: macOS, Linux (в настоящее время поддерживает Debian 9 и Ubuntu 18.04 LTS) или любой хост с Docker

Примечание: В настоящий момент официальная поддержка Windows отсутствует. Однако участники сообщества смогли установить ноды на Windows Subsystem для Linux. Нажмите [здесь](https://minawiki.com/index.php?title=Mina_on_Windows), чтобы ознакомиться с инструкциями по использованию Windows, разработанными комьюнити. Пользователи Windows могут также использовать документацию Docker.

**Аппаратное обеспечение**: Для отправки и получения mina не требуется специальное оборудование, но для запуска блок-продюсера в сети Mina требуется следующее:

- минимально 8-ядерный процессор
- не менее 16 ГБ оперативной памяти

Внимание: если вы планируете запускать ноду snark-воркер и блок-продюсер одновременно, вам может потребоваться больше оперативной памяти. Сейчас не требуется использование графических процессоров, но они могут понадобиться операторам нод после обновления протокола.

**Соединение сети**: не менее 1 Мбит/с

**Виды виртуальных машин (VM)**:
O(1) Labs протестировали работу нод на нескольких облачных провайдерах и рекомендуют операторам нод использовать следующие виртуальные машины с необходимыми характеристиками. Примите во внимание, что вы можете использовать другие VM в зависимости от ваших индивидуальных требований или ценовой категории.

- AWS - [c5.2xlarge](https://www.ec2instances.info/?filter=c5.2xl&region=us-west-2&cost_duration=daily&selected=c5.2xlarge)
- GCP - [c2-standard-8](https://cloud.google.com/compute/docs/machine-types)
- Azure - [Standard_F8s_v2](https://docs.microsoft.com/en-us/azure/virtual-machines/windows/sizes-compute#fsv2-series-1)
- Digital Ocean - [c-8-16gib](https://cloud.digitalocean.com/droplets/new?size=c-8-16gib)

Ознакомьтесь с нашим [руководством](https://youtu.be/M9lE2lgyVEo), чтобы узнать, как настраивать Mina в Google Cloud.

### Ubuntu 18.04 / Debian 9

Добавьте репозиторий Mina Debian и установите:

```
echo "deb [trusted=yes] http://packages.o1test.net release main" | sudo tee /etc/apt/sources.list.d/mina.list
sudo apt-get update
sudo apt-get install -y curl unzip mina-mainnet=1.1.8-b10c0e3
```

После запуска `mina version`. убедитесь, что daemon установлен правильно. Должно появиться следующее: `Commit b10c0e3db9112a2a8aebc3eec7c6d2570fcc4044 on branch master`.

## Настройка переадресации портов и любых файрволов

Если вы используете файрвол, вы должны разрешить трафик на TCP-порт 8302. Кроме того, если не указан флаг `-external-ip YOUR_IP`, daemon будет использовать HTTPS (443) и HTTP (80), чтобы попытаться определить свой собственный IP-адрес.

Возможно, потребуется настроить переадресацию портов вашего маршрутизатора, чтобы разрешить входящий трафик на следующие порты через **внешний** IP-адрес.

- `TCP` port `8302`
  _Опционально_ при использовании службы GraphQL откройте `TCP` порт `3085`

Смотрите подробные инструкции в <DocLink copy="этом руководстве" url="/troubleshooting#port-forwarding" />.
