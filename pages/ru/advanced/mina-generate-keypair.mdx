# Создание keypair mina

Чтобы упростить процесс генерации ключей, мы создали простую утилиту с командной строкой под названием `mina-generate-keypair`. Инструмент `mina-generate-keypair` имеет простой функционал для создания вашего keypair, позволяющий участвовать в сети Mina. Ниже описано, как установить и использовать инструмент в вашей локальной системе.

<br />

<Alert kind="warning">
  Никогда не используйте одну и ту же пару ключей из тестовой сети в mainnet!
</Alert>

## Установка

Мы предоставляем поддержку `mina-generate-keypair` для нескольких операционных систем. Следуйте инструкциям ниже, чтобы установить инструмент на вашу OC.

### macOS

Если вы используете macOS, установите Docker и следуйте приведенным ниже инструкциям.

### Ubuntu 18.04 / Debian 9

После <DocLink copy="добавления репозитория Mina" url="/getting-started#ubuntu-1804--debian-9" /> , вы можете просто выполнить следующую команду.

```
echo "deb [trusted=yes] http://packages.o1test.net release main" | sudo tee /etc/apt/sources.list.d/mina.list
sudo apt-get update
sudo apt-get install mina-generate-keypair=1.1.8-b10c0e3
```

Убедитесь, что инструмент для генерации ключей установлен правильно, запустив `mina-generate-keypair -version`. Вы должны получить `Commit b10c0e3db9112a2a8aebc3eec7c6d2570fcc4044 on branch master` на выходе.

### Windows / Другие платформы

Установите Docker и следуйте инструкциям, указанным в пункте 3b ниже.

## Использование

Создание пары ключей выполняется в несколько простых шагов.

1. Сначала убедитесь, что в вашей системе есть папка для хранения файлов с ключами. Мы рекомендуем использовать папку `~/keys`.

```
mkdir ~/keys
```

2. Затем убедитесь, что разрешения для этой папки установлены правильно, это предотвратит доступ к файлам нежелательных процессов.

```
chmod 700 ~/keys
```

3. Создайте ключ, используя инструкции в пункте 3a для Debian/Ubuntu или пункте 3b для Docker.

<br />

<Alert kind="warning">

Обязательно установите новый надежный пароль для следующих команд. Mina никогда не будет запрашивать у вас этот пароль, и вы не должны никому его сообщать.

</Alert>

<br />

3a. Для Ubuntu/Debian: Создайте ключи с помощью команды `mina-generate-keypair`. При появлении запроса введите пароль, который вы хотите использовать для защиты ключей. Если он уже установлен, инструмент будет использовать пароль из переменных среды CODA_PRIVKEY_PASS. <em>НЕ ЗАБУДЬТЕ этот пароль.</em>

```
mina-generate-keypair -privkey-path ~/keys/my-wallet
```

3b. В Docker на Windows/MacOS/Linux: сгенерируйте ключи, используя образ Docker `minaprotocol/generate-keypair`. При появлении запроса введите пароль, который вы хотите использовать для защиты ключей. <em>НЕ ЗАБУДЬТЕ этот пароль.</em>

```
cd ~
docker run  --interactive --tty --rm --volume $(pwd)/keys:/keys minaprotocol/generate-keypair:1.0.2-06f3c5c -privkey-path /keys/my-wallet
```

В вашей системе будут созданы два файла: `~/keys/my-wallet`, содержащий зашифрованный закрытый ключ, и `~/keys/my-wallet.pub`, который содержит открытый ключ в виде обычного текста. Пожалуйста, сохраните файл с закрытым ключом и пароль, который вы использовали, в надежном месте, например в менеджере паролей.

<br />

<Alert kind="warning">

Никогда не делитесь своим закрытым ключом и убедитесь в надежности его хранения. Если вы потеряете свой закрытый ключ, или его получат злоумышленники, вы потеряете доступ к своему аккаунту и средствам на нем. Вместо этого всегда делителсь тоолько своими открытыми ключами. Mina никогда не станет запрашивать у вас ваши закрытые ключи.

</Alert>

<br />

4. Наконец, убедитесь в правильности установленных разрешений для файла закрытого ключа, это предотвратит доступ к нему нежелательных процессов.

```
chmod 600 ~/keys/my-wallet
```

## Подтвердите ваш закрытый ключ

После того, как вы создали ключ, вам нужно убедиться, что он работает. Достаточно убедиться, что вы можете подписать транзакцию. Вы можете проверить это с помощью `mina-validate-keypair`.

На Linux выполните следующую команду:

```
mina-validate-keypair -privkey-path <path-to-the-private-key-file>
```

При использовании Docker выполните следующую команду:

```
docker run --interactive --tty --rm --entrypoint=mina-validate-keypair --volume $(pwd)/keys:/keys minaprotocol/generate-keypair:1.0.2-06f3c5c -privkey-path /keys/my-wallet
```
