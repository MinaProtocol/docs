# Токены

У пользователей есть возможность создавать и выпускать собственные пользовательские токены, для чего требуется специальный токен-аккаунт.

<Alert kind="warning">

Начиная с версии `0.0.16-beta7` эта функция была отключена и может быть использована только при включении флага сборки. Мы планируем активировать эту возможность в будущих релизах.

</Alert>

В протоколе Mina пользователи могут создавать свои собственные Токены, которые могут быть созданы и использованы с помощью специальных токен-аккаунтов. Mina CLI (интерфейс командной строки) - это основной способ взаимодействия с токенами на блокчейне Mina. Он предоставляет клиентский интерфейс, поддерживающий функционал для выпуска новых токенов, создания новых токен-аккаунтов, а также создания нон-дефолтных токенов. В дополнение к интерфейсу токенов, есть и другие продвинутые клиентские и daemon-команды (см. <DocLink copy="Ссылка CLI" url="/cli-reference#cli-reference" />).

## Создание токенов

С помощью CLI вы можете создавать свои собственные токены на Mina. Нон-дефолтные токены - это новый тип токенов, который вводится в блокчейн Mina. Данный вид токенов имеет уникальный идентификатор, который отличает их от всех остальных токенов в блокчейне. Пользователи с соответствующими токен-аккаунтами могут получать/отправлять этот новый тип токена, который полностью отделен от самого токена Mina! Когда вы создаете новый токен, вы можете затем получать еще больше уже существующих токенов (см. ниже в разделе <DocLink copy="Minting Tokens" url="/tokens#minting-tokens" />).

При создании нового токена создается новый ID токена, а также токен-аккаунт для Вашего открытого ключа. Этот токен-аккаунт становится владельцем токена и, за счет этого, может выпускать больше токенов. Аккаунт также (в конечном счете) имеет право включать/отключать учетные записи, а также устанавливать, могут ли создаваться новые аккаунты для токена.

Обратите внимание, что создание новых токенов требует оплаты дополнительных сборов наряду с постоянно действующими комиссиями за транзакции. Эта дополнительная комиссия подлежит оплате при создании нового токен-аккаунта. Это означает, что оплата за создание счета взимается в дополнение к комиссии за транзакции, которая взимается в обычном порядке.

### Создание нового токена

```
$ mina client create-token -sender <PUBLIC_KEY>
```

<Alert>

Требуемое поле для создания нового токен-аккаунта:

- -** отправитель** : Открытый ключ, с которого вы хотите совершить транзакцию

</Alert>

Посмотреть

```
$ mina client create-token -help
```

чтобы узнать больше о том, как использовать команду.

## Создание токен-аккаунта

С помощью CLI вы можете создать токен-аккаунт для уже существующих токенов. Эти аккаунты похожи на обычные учетные записи, но предназначены исключительно для взаимодействия с нон-дефолтными токенами, которые существуют в протоколе Mina. Вы должны создать отдельный уникальный токен-аккаунт для каждого типа токенов, с которым вы хотите взаимодействовать.

<Alert kind="warning">

Чтобы отправить или получить существующий токен, у вас должен быть токен-аккаунт с указанным токеном. В случае отсутствия действующего токен-аккаунта с данным токеном, сделка не может быть совершена.

</Alert>

### Создание нового токен-аккаунта

```
$ mina client create-token-account -token-owner <PUBLICKEY> -receiver <PUBLICKEY> -sender <KEY> -token <TOKEN_ID>
```

<Alert>

Если владелец токена не указан, CLI сделает запрос на поиск владельца токена. Если владелец не будет найден, команда выдаст ошибку.

Поля, обязательные для создания нового токен-аккаунта:

- -**получатель** : Публичный ключ для создания новой учетной записи, вы можете повторно использовать ваш обычный публичный ключ здесь

- -** отправитель** : Открытый ключ, для которого вы хотите создать учетную запись

- -**токен** : Идентификатор токена, с которого вы хотите совершить транзакцию

</Alert>

Посмотреть

```
$ mina client create-token-account -help
```

чтобы узнать больше о том, как использовать команду.

## Минтинг токенов

С помощью CLI вы можете эмитировать собственные токены. Выпуск собственных токенов просто увеличивает запас определенного токена. Эмиссия токенов - это то, как создаются все нон-дефолтные токены. Нет никаких других протокольных событий или команд для этого. Данная команда может быть выдана только владельцем токена, но токены могут быть эмитированы любым существующим аккаунтом для этого токена.

### Минтинг токена

```
$ mina client mint-tokens -amount <VALUE> -sender <PUBLIC-KEY> -token <TOKEN_ID>
```

<Alert>

Поля, обязательные для создания нового токен-аккаунта:

- -**сумма** : Количество новых токенов для создания

- -** отправитель** : Открытый ключ, с которого вы хотите отправить транзакцию

- -**токен** :ID токена для эмиссии

Существует также необязательное поле, в котором можно указать открытый ключ учетной записи для создания новых токенов:

- -**получатель** : Публичный ключ учетной записи для создания новых токенов в

</Alert>

Посмотреть

```
$ mina client mint-tokens -help
```

чтобы узнать больше о том, как использовать команду.
