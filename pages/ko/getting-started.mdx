# 시작하기

미나에 오신것을 환영합니다! 이번 섹션은 미나 노드를 설정하는 방법을 알려드리겠습니다. 우선, 네트워크에서 사용할 키 쌍을 어떻게 생성하는지 설명드릴 것입니다. 최종적으로 네트워크에 어떻게 접속하고, 블록 생산을 시작하는 방법을 설명해 드리겠습니다.

<Alert>

이 문서는 **메인넷** 전용입니다.

네트워크 상태: <StatusBadge service="Network" />

</Alert>

## 요구 사항

**소프트웨어**: macOS, Linux (현재는 Debian 9,  Ubuntu 18.04 LTS 배포판을 정식 지원), 혹은 Docker 구동이 가능한 환경

참고: Windows 는 현재 공식적으로 지원되지 않습니다. 하지만, 커뮤니티 회원들이 Windows Subsystem for Linux 를 통해 성공적으로 노드를 구성하는데 성공했습니다. [여기](https://minawiki.com/index.php?title=Mina_on_Windows)에서 커뮤니티 회원이 발견한 Windows 환경을 기반한 설명을 보실 수 있습니다. 추가적으로, Windows 사용자들은 Docker 를 사용하는 문서를 보고 따라하셔도 됩니다.

**하드웨어**: 미나를 보내고 받는데에는 특별한 하드웨어 요구 사항이 없지만, 블록 생산을 위한 노드 구성에는 아래와 같은 요구 사항이 필요합니다:

- 8코어 이상의 프로세서
- 16GB 이상의 메모리

혹시 스나크 채굴 및 블록 생산을 동시에 하는 노드를 구동하고자 한다면, 더 많은 메모리가 필요할 수 있음을 참고하시기 바랍니다. 그래픽카드는 아직까지 필요없지만, 프로토콜이 업그레이드 되면서 필요하게 될 수도 있습니다.

**네트워크**: 1 Mbps 속도 이상의 네트워크 환경

**가상 인스턴스**:
O(1) Labs 는 특정 클라우드 플랫폼들을 통해 노드를 구동하는것을 테스트 해봤으며, 아래와 같은 인스턴스 사양을 사용해서 가장 기본적인 노드를 구동하는것을 추천합니다. 사용자 개별의 요구 사항들 및 다양한 비용 제약 조건에 따라 인스턴스 유형이 달라질 수 있음을 명심하시기 바랍니다.

- AWS - [c5.2xlarge](https://www.ec2instances.info/?filter=c5.2xl&region=us-west-2&cost_duration=daily&selected=c5.2xlarge)
- GCP - [c2-standard-8](https://cloud.google.com/compute/docs/machine-types)
- Azure - [Standard_F8s_v2](https://docs.microsoft.com/en-us/azure/virtual-machines/windows/sizes-compute#fsv2-series-1)
- Digital Ocean - [c-8-16gib](https://cloud.digitalocean.com/droplets/new?size=c-8-16gib)

저희의 [가이드](https://youtu.be/M9lE2lgyVEo)를 통해 구글 클라우드를 통해 미나 노드를 구성하는 방법을 배워보실 수 있습니다.

## 설치

최신 릴리즈된 바이너리는 아래에서 찾을 수 있습니다. 아래에 macOS 및 Linux 를 위한 설치 방법이 나와있습니다.

바이너리 크기가 1GB 정도 되므로, 설치에 시간이 걸릴 수 있습니다.

<Alert kind="warning">

만약 이전 버전의 `mina` 를 설치하셨다면, 최신버전으로 업그레이드 하셔야 합니다. 그렇지 않으면 오래된 클라이언트로 간주되어 네트워크에서 차단당할 수 있습니다. 아래에서 macOS 및 Linux 의 빌드를 업그레이드 할 수 있는 방법이 나와있습니다.

</Alert>

### Ubuntu 18.04 / Debian 9

미나 Debian 저장소를 추가하고 설치합니다

```
echo "deb [trusted=yes] http://packages.o1test.net release main" | sudo tee /etc/apt/sources.list.d/mina.list
sudo apt-get update
sudo apt-get install -y curl unzip mina-mainnet=1.1.5-a42bdee
```

`mina version` 을 실행하여 데몬이 올바르게 설치됐는지 확인하세요. 출력 결과는 다음과 같아야 합니다 `Commit a42bdeef6b0c15ee34616e4df76c882b0c5c7c2a on branch master`.

### Windows

윈도우 환경은 내부적으로 지원이 되지 않습니다. 혹시 Windows 환경에서 미나를 개발하기 위한 보조를 받고 싶으시다면, grants@o1labs.org 로 이메일을 보내시거나, [디스코드 서버](https://bit.ly/MinaDiscord)에 들러보시기 바랍니다.

아래의 Docker 환경에서의 설치 방법을 참고하시기 바랍니다.

### macOS

macOS 는 개발자들이 사용하고는 있지만, 공식적으로 아직까지는 미리 빌드된 패키지들을 제공하지 않습니다.

아래에서 Docker 환경에서의 설치 방법을 참고하거나, 소스를 통해 직접 빌드하는 방법을 참고하시기 바랍니다.

### Docker

Docker 는 필요한 소프트웨어를 실행할 가상 머신을 구동해주는 크로스플랫폼 도구입니다. [이 페이지](https://docs.docker.com/get-docker/)에서 환경에 맞는 설치방법을 참고해주시기 바랍니다. Docker 설치가 완료됐다면, <DocLink copy="네트워크에 연결하기" url="/using-mina/connecting" /> 페이지로 이동해서 계속 진행하시기 바랍니다.

### 소스를 통해 빌드하기

혹시 지원하지 않는 Linux 배포판이나 macOS 버전을 사용하고 계시는경우, [소스 코드를 통한 미나 빌드](https://github.com/MinaProtocol/mina/blob/master/README-dev.md#building-mina)를 시도해보시기 바랍니다. 이밖에 다른 운영체제에서는 구동 테스트가 완전히 진행되지 않았고, 문제가 있을수도 있음을 명심하시기 바랍니다. 혹시 과정상 발생한 로그나 문제해결에 도움이 필요하시다면 [디스코드 서버](https://bit.ly/MinaDiscord) 에 방문해서 공유해주시기 바랍니다.

## 포트포워딩 및 방화벽 설정

방화벽을 사용중이시면, TCP 8302 포트에 대한 허가가 필요합니다. 추가적으로, `-external-ip YOUR_IP` 플래그가 주어지지 않는이상, 데몬이 클라이언트의 IP 주소를 찾는 과정중에 HTTPS (443) 혹은 HTTP (80) 를 사용하게 됩니다.

사용하고 계시는 라우터의 포트포워딩 설정을 통해 **외부** IP 주소의 다음포트들의 인바운드 트래픽을 허용시켜야 할 수도 있습니다.

- `TCP` 의 `8302` 포트
- _선택적으로_ GraphQL 서비스를 사용하시려면 `TCP` 의 `3085` 포트또한 포워딩이 필요합니다

단계별 설명은 <DocLink copy="해당 가이드" url="/troubleshooting#port-forwarding" />를 참고하시기 바랍니다.
